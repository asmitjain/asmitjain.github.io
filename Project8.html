SETUP:

Once you gather all the hardware and software, your very step should be to set up your RPi. This section will give in-depth instructions on how to install all the required software from scratch. If in case your Pi crashes, you can follow the same instructions to troubleshoot.

For a new device, you need a Pi with pre-installed Raspbian. Most of the devices come with the installed software. You just need to connect the power cord. If you don’t have the pre-installed file, you can download files from this website. Select Offline & Network Install and download. You might need to format the SD card first. For formatting the card and in-depth installation, follow this video

Note: In case of a crashed Pi, you have two options. The first one is to completely format the SD card and reinstall the Raspbian on the SD card. For this, follow the instructions given in the previous paragraph. The second option is to use the old Raspbian already installed on the crashed SD card. This is not a recommended step as the installation files, memory partition, or some other system files might be corrupt. Use this option only if you don’t have any source to format the SD card and download the setup.

Getting Started: An easy way is presented in this video

For getting started, connect HDMI, mouse and keyboard to the Pi. Also, plug a network cable. After a colorful screen, a setup page will appear automatically. Select Raspbian and hit install. If you are using the old hardware, press shift (when it gives the option) to open the recovery mode. Select Raspbian and hit install. A confirmation page will appear, which will warn you about your data loss. Wait for 30 min to install the OS.

1. Desktop Menu: After you get to the desktop, the next thing we have do is to set up the software part. Start with configuring the Pi. Watch the previous video to do the same. After configuration, it will ask for a reboot. Note: If the time is not updated, try to check the LAN connection or set WiFi, if available. As soon as the connection is established, the date and time will get updated.

2. Software: The Pi comes with preinstalled apps that take up a lot of space. The first thing we can do is to uninstall all of them to save some memory. Open the terminal by clicking the terminal icon on the top ribbon. See attached picture.

In the terminal, write “sudo apt-get purge -y packagename”. This command removes the software from the Pi. Below is the list of all the packages. If you don’t want to write commands again and again. You can write all the packages in just one line. For example, “sudo apt-get purge -y packagename1 packagename2 packagename3 packagename4…..”. The list of all the packages are listed below:

wolfram-engine, scratch, scratch2, claws-mail, smartsim, micecraft-pi, geany, bluej, sonic-pi, nodered greenfoot, python-pygame

Updating the package database requires superuser privileges, so you’ll need to use sudo. After removing the software, write sudo apt-get update. Once you have updated the package database, you can upgrade the installed packages sudo apt-get upgrade. This might take some time. For all the other details, follow this video. You can use the command below to clean the local repository of retrieved package files and save some space sudo apt-get clean

One of the many ways is to use autoclean. Unlike the above clean command, autoclean only removes those retrieved package files that have a newer version now, and so won’t be used anymore sudo apt-get autoclean. Another way to free up disk space is to use autoremove. It removes libraries and packages that were installed automatically to satisfy the dependencies of another installed package. If that package is removed, these automatically installed packages are useless in the system. This command removes such packages. sudo apt-get autoremove.

To check the space on RPi, type df -h To clear the content on the screen, use clear

The next thing you can do is to download TeamViewer. This will allow you to control your Pi remotely. TeamViewer The default user ID is pi and the password for Raspberry Pi is raspberry After installation, open TeamViewer Host and click on the setting option (gear icon). Go to security and assign a password. Note down your unique ID so that you can control it remotely. Now, login to the computer which has all the backup files. Open TeamViewer on that and write RPi’s ID to connect the system. Once done, you will find an option in the top ribbon Transfer Files. Locate the files on the backup computer and hit the SEND/RECEIVE button to transfer files.

3. Packages: To use the Labjack data loggers, you will need some files which will help us to establish a connection between the Pi, Python and Labjack. The first step is to install the LJM Software Installers. Open LabJack LJM. Look for Linux ARMv7. If you are using a different OS (not on RPi), you can select the respective downloadable file. Download the release version –LabJackM-1.1804-Raspbian-Linux-armhf-release.tar.gz. This is a zip file (just like windows). The next step is to unzip it. There are two options to do that. Either you can open the folder and use the EXTRACT option on the top ribbon. Or you can simply write the commands mentioned below. But first, make sure that you are in the location where the zip file is. For example, if the file is in the Download folder and you are on Desktop. You will have to write cd Downloads in the terminal. Once you are at the right location, write:

$ tar -zxvf LabJackM-1.1804-Raspbian-Linux-armhf-release.tar.gz
$ cd LabJackM-1.1804-Raspbian-Linux-armhf
$ sudo ./LabJackM.run

This installation was for RPi and Labjack. Now, if we are using Python script to control the device, we will need to install Python LJM. You can either write: sudo pip install labjack-ljm in the terminal. Or you can download the files from the link mentioned on LabJack Python . After downloading, unzip the folder and run the setup file.

You are all set! This completes your installation. You can now run your script.

WORKING CONDITIONS:

• There are two main cycles in the system: The first one is the Cycle1 which drives one type of operation and the second one is the Cycle2, which performs the other type of operation. The current code is set to perform the cycle1 for x min and the cycle2 for y min. The flexibility is given to the customers where they can change the timing.

• As soon as the code is run, it starts recording the system parameters. These parameters are recording the data at an interval set by the customer.

• It performs some complex calculations and notify the customer if they are out of bounds.

• After performing calculations and comparing data to set parameters, it will automatically decide what values are acceptable. And based on that, it will continue the cycle1. If the values are out of bounds, it will perform the cycle. On the display, the customer can see the parameters along with the system current status (cycle1/ cycle2)

• The code won’t do the calculations until it collects the n values. Once it collects and performs calculations, it will automatically clear the list and put new data.

• If consecutive cycles of cycle2 are occurring, it keeps count of it. If there are more than n consecutive cleaning cycles, it will display a message “Maintenance revery quired” along with the current cycle type. It indicates that there is something wrong with the system components. The maintenance message won’t clear until the team fixes the system.

• All the data can be displayed on the code terminal, dashboard, mobile app in real-time. The push API will send the data to the cloud in JSON format. And it will always save the .CSV log file in the system too if there is any problem with the internet data transfer. We can add/ delete header columns for different values. It also gives the date and time stamp to all the data.

• The parameters calculated were related to electrical properties of the internal components, system status, date/ time, maintenance count, number of data points, etc.

There are several timers which initiate during the process to track different activities. They all get triggered at different points and reset itself after the set activities have been performed.
